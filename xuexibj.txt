	
GIT 学习笔记 2016-02-02

本次注册自己的账号
git config --global user.name ""
git config --global user.email ""

创建目录
cd e:
mkdir mygitfile
cd mygitfile

在当前目录中建立版本库，这个时候文件中会多出一个.git的文件
git init

添加文件至缓冲区
git add readme.txt

提交文件至版本库中
git commit -m "描述"


如果没有输入"-m"系统会进入VIM的编辑状态进行描述的输入。
git commit 提交命令一次可以提交多个文件

git commit -am "描述"


git status
了解当前仓库的状态，很常用


git log
可以检查文件修改日志


git reset --hard head^ 可以回滚到上一个版本

git reset --hard head~100 回滚前100个版本


如果回滚后发现回滚错误，可以通过ID返回，通过LOG中的ID
git reset --hard ID

git reflog 提供了每次文件回滚和提交的记录

git rm readme.txt 提交删除文件的信息到缓冲区,提交后同样使用commit提交

二、学习建立远程仓库

生成SSH
$ ssh-keygen -t rsa -C "youremail@example.com"

关联远程库
git remote add origin git@github.com:yellowkui/mygitfile.git
git remote add origin https://github.com/yellowkui/mygitfile.git

如果上面关联错了通过下面语句删除
git remote rm origin

把本地库内容推送至远程库
git push -u origin master

首次将数据推送至远程库时需要加 -u的参数
git push -u origin master

以后推送时，只需要下面语句即可
git push origin master


远程克隆命令，有两种协议一种SSH协议，一种是HTTPS协议
git clone git@github.com:yellowkui/newWorld.git


三、分支操作

建立一个新分支newBranch,并指向这个分支
git checkout -b "newBranch"

上面的命令也可以通过实现
git branch "newBranch"
git checkout "newBranch"

查询现在的分支
git branch

删除分支
git branch -b "newBranch"

四、合并分支
首先指向到master分支
git checkout master

合并分支的语句 ，把某分支合并至当前的分支
git merge "newBranch"
